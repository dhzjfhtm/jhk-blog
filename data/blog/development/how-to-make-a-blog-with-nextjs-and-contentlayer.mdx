# Tailwind Nextjs Starter Blog 사용하기

Fork 한 Github 저장소: [Tailwind Nextjs Starter Blog](https://github.com/timlrx/tailwind-nextjs-starter-blog)

## Next.js

Next.js는 React 기반의 프레임워크로 SSR(Server Side Rendering)을 지원합니다.

### Next.js를 선택하는 이유

1. **SEO 친화적**: SSR 덕분에 블로그가 검색 엔진에 더 최적화됩니다.
2. **빠른 로딩 시간**: 사전 렌더링으로 사용자가 콘텐츠를 더 빨리 볼 수 있습니다.
3. **풍부한 기능**: API 라우트, 자동 코드 분할, 내장 CSS 지원 등의 기능이 있습니다.

## Tailwindcss

Tailwindcss는 유틸리티 중심의 CSS 프레임워크입니다. 커스텀 CSS를 작성하는 대신에 미리 정의된 클래스를 HTML 요소에 적용할 수 있습니다.

### Tailwindcss 사용의 장점

1. **빠른 개발**: 충돌 걱정 없이 빠르게 블로그를 만들고 스타일링할 수 있습니다.
2. **커스터마이징 용이**: 설정을 쉽게 조정하여 디자인을 원하는 대로 만들 수 있습니다.
3. **효율적**: 클래스를 조합하여 중복되는 CSS를 줄일 수 있습니다.

## Contentlayer

Contentlayer는 Next.js와 Tailwindcss를 이용하여 블로그를 만들 때 사용할 수 있는 라이브러리입니다. 이는 마크다운/MDX 파일과 Next.js 페이지 사이의 데이터 계층 역할을 합니다.

### Contentlayer 사용 이유

1. **Markdown 지원**: Markdown 또는 MDX로 블로그를 쓸 수 있어 콘텐츠에 집중하기 쉽습니다.
2. **동적 콘텐츠**: 빌드 시간이나 런타임에 데이터를 가져와 블로그를 더 동적으로 만들 수 있습니다.
3. **유연한 스키마**: 스키마를 통해 콘텐츠의 형태를 정의하면 쿼리하기 쉽습니다.

Next.js, Tailwindcss, Contentlayer를 사용하여 블로그를 만드는 것은 온라인에 자신의 생각을 게시하는 효율적이고 효과적인 방법입니다. 이 스택은 깊은 설정이나 코드 작업 없이도 빠르고, SEO 친화적이며, 아름다운 블로그를 만들 수 있는 유연성을 제공합니다.
